我是一边学习刘宇波老师的算法课程，同时完成相应题目的训练的。因此我会在readme中总结刘宇波老师提到的常用的解决算法问题的思路，并给出相对应的leetcode题目解法，希望对大家有所帮助。
## 解决算法面试问题的整体思路
- 注意题目中的条件
- 没有思路的时候
  - 自己给自己几个简单的测试用例，实验一下。
  - 不要忽视暴力解法。
- 优化算法
  - 遍历常见算法思路，看是否适用这个问题
  - 遍历常见的数据结构
  - 空间和时间的交换（使用哈希表）
  - 预处理信息（排序）
  - 在瓶颈处寻找答案
- 实际编写
  - 极端条件的判断
  - 变量名
  - 模块名，复用性
- 不要忽视基本的算法和数据结构
## 面试中的时间复杂度分析
- 算法复杂度在有些情况下是和用例相关的
- 数据规模的概念
  - 1s内解决问题的话o（n^2）可以处理10^4级别的数据
  - 1s内解决问题的话o（n）可以处理10^8级别的数据
  - 1s内解决问题的话o（nlogn）可以处理10^7级别的数据
  - 以上想要更稳妥的话除以10
- 空间复杂度
  - 递归调用是有空间代价的，递归的深度有多深，空间复杂度就是多少
- 递归算法的复杂度分析
  - 递归中进行一次复杂度调用
   - 如果递归函数中只进行一次递归调用，递归深度为depth，在每个递归函数中时间复杂度为T，则总体的时间复杂度为o（T*depth）
 - 递归中进行多次递归调用
   - 计算调用的次数
- 均摊复杂度分析
  - vector的push_back操作
- 避免复杂度的震荡
